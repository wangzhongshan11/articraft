<link_naming>
<!-- Link 命名规范：link 名称本身属于交付物质量门槛的一部分 -->

- **Link 名称**是交付物质量门槛的一部分：保持**简洁、语义化**，并基于物体**固有坐标系（intrinsic frame）**命名，而非任意标签或**状态型**标签。
- 为每个 link 赋予**极简语义名**：理想情况下仅为**零件名**；仅当需要区分相似零件时，才附加简短的**固有位置**或**形状提示**。
- 每个 link 名称必须是**单条**以下划线连接的字符串，**最多 5 个词（words）**。
- **不要**在 link 名称中编码关节**状态**。避免状态词，例如：`open`、`closed`、`extended`、`pulled_out`、`ajar`、`tilted`、`rotated` 等。
- 优先使用能说明**“该零件是什么”**的名称；在有帮助时可说明**形状**。
- 仅当物体具有**有意义的规范朝向**，或其他清晰的**物体固有参考系**时，才使用**位置词**。
- 当相似零件可可靠区分时，优先使用物体固有空间提示，例如：`front_handle`、`side_support`。
- **不要**对对称或朝向模糊的物体臆造 `left`、`right`、`front`、`back` 区分。部分物体仅有**部分**固有坐标系：人形部件可用 `left_arm`；但对称柜体的两扇门通常**不应**命名为 `left_door` 与 `right_door`。
- 若仅有部分固有坐标系有意义，则**只使用有意义的那部分**。例如：若 `front` 与 `back` 有意义而 `left`/`right` 模糊，则在需要时使用 `front_*` 或 `rear_*`，**不要**强行加侧面标签。
- 若重复零件在语义上相同且**无法在固有意义上区分**，则复用同一基础名并加数字后缀，例如：`door_0`、`door_1`。对于二维重复布局，可使用 `key_0_0`、`key_0_1` 等形式。
</link_naming>
